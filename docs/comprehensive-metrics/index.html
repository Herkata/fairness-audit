Step 3: Check Base Rate Differencesâ€‹

If groups have very different base rates, parity on outcomes may conflict with accuracyâ€”favor error-based metrics and calibration.

Step 4: Evaluate Intersectional Riskâ€‹

If diverse population or layered discrimination: include Multi-Attribute Error Rate Balance and worst-group checks.

Step 5: Consider Legal & Regulatory Constraintsâ€‹

Align metrics with domain expectations (e.g., disparate impact in lending, adverse impact ratios in employment).

Step 6: Decide Based on Error Asymmetryâ€‹

Prioritize the error type that causes greater harm (e.g., false negatives in lending/healthcare, false positives in criminal justice).

Step 7: Always Combine Metricsâ€‹

Minimum package:

  • One outcome distribution metric (e.g., Demographic Parity)
  • One error-based metric (Equal Opportunity or Equalized Odds)
  • One intersectional metric
  • Statistical validation layer

Notes (selection decisions)â€‹

  • Record the decision type, primary harm, and whether base-rate gaps exist.
  • Capture why specific metrics were included or excluded (e.g., legal alignment, stakeholder ask).
  • Note data caveats (imbalance, proxies, missing protected attributes) and mitigation steps.
  • Link to the notebook/dashboard and model version used for metric calculations.
